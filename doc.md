### 1. Концепция игры: "Geometry Flow" (Рабочее название)

Вместо того чтобы просто копировать Vampire Survivors, мы изменим способ взаимодействия игрока с оружием.

**Киллер-фича: "Конструктор Заклинаний" (Modular Projectile Synthesis)**

В обычных "выживачах" ты берешь "Топор" и он летит по дуге. В твоей игре ты собираешь **алгоритм поведения** снаряда.

*   **Визуал:** Игрок — это абстрактное светящееся ядро (Core). Враги — геометрические фигуры (треугольники быстрые, квадраты медленные, пятиугольники стреляют).
*   **Геймплей:**
    1.  Персонаж стреляет автоматически базовыми частицами.
    2.  При повышении уровня вы не выбираете "Новое оружие", вы выбираете **Модификатор потока**.
    3.  У игрока есть слоты (например, 3 канала атаки).
    4.  **Пример сборки:**
        *   Слот 1: [Генератор (Пуля)] -> [Модификатор (Разделение при ударе)] -> [Эффект (Взрыв)].
        *   Слот 2: [Генератор (Луч)] -> [Модификатор (Отражение от стен)] -> [Эффект (Заморозка)].

**Почему это круто (MVP версия):**
Вам не нужно кодить 50 разных пушек. Вы кодите 3 типа генераторов (пуля, луч, аура) и 5 модификаторов (скорость, размер, рикошет, орбита, самонаведение). Их комбинации создают сотни вариаций. Это создает глубину (Theorycrafting), которую любят хардкорные игроки.

**Абстрактный стиль:**
*   **Фон:** Темно-синий/черный (Hex #050510).
*   **Враги:** Неоновые контуры с `Bloom` (свечением).
*   **Урон:** Вспышки белого, экран слегка трясется (Screen Shake) при критах.
*   **Смерть врагов:** Они не оставляют трупы, а распадаются на партиклы (используем `bevy_hanabi`).

### 2. Техническая реализация на Bevy (MVP за неделю)

Bevy идеально подходит, так как он Data-Driven.

#### Стек и ассеты:
*   **Движок:** Bevy 0.15 (используй последнюю стабильную версию).
*   **Графика:** `bevy_prototype_lyon` (для рисования векторов/форм без текстур) или процедурные меши. Лучше `Sprite` с простыми белыми текстурами круга/квадрата, которые ты красишь через `Color` компонента — это дешевле для рендеринга.
*   **Эффекты:** `bevy_hanabi` (GPU партиклы — мастхэв для "дорогого" вида).
*   **Звук:** Генерируй SFX через `sfxr` (retro style) или возьми бесплатные абстрактные паки (Sonnis GDC bundles).
*   **UI:** Стандартный `bevy_ui`.

#### Архитектура (ECS):

1.  **Components:**
    *   `Player`, `Enemy`, `XpOrb`.
    *   `Health`, `Velocity`, `Damage`.
    *   `Projectile` — содержит в себе граф поведения (список enum-ов эффектов).
2.  **Systems:**
    *   `movement_system`: двигает всё, что имеет Velocity.
    *   `collision_system`: для MVP не бери `avian` или `rapier` (сложно настраивать физику за неделю). Напиши простую Spatial Hash Grid или просто проверку дистанции (`distance_squared`), так как все объекты — круги/квадраты. Это сэкономит кучу времени.
    *   `spell_system`: итерируется по оружию игрока, спавнит снаряды.
    *   `behavior_system`: управляет логикой снарядов (если есть самонаведение — меняет вектор скорости).

#### Шейдеры (Грааль визуала):
Вам не нужны ассеты, вам нужен **Post-Processing**.
Включи в Bevy `Tonemapping::TonyMcMapface` и **Bloom**.
Простой белый круг с `Color::rgb(2.0, 0.5, 0.5)` (значения > 1.0) при включенном Bloom даст мощный неоновый эффект. Это "продаст" картинку.

### 3. Монетизация (Web / Yandex Games)

Так как Steam отпадает, а Google Play сложен с выплатами, **Яндекс Игры** — идеальный вариант для MVP. Они дают органический трафик и платят на карты РБ (через партнеров или напрямую, если правила позволяют, либо через крипту/посредников, но технически — это веб).

**Модель:** Hybrid (Rewarded Ads + Interstitial). IAP (внутриигровые покупки) для MVP за неделю делать долго.

1.  **Interstitial (Межстраничная реклама):**
    *   Показывать *только* после смерти или победы. Не прерывай геймплей.
2.  **Rewarded Video (Видео за вознаграждение) — Самое важное:**
    *   **Воскрешение:** Один раз за забег возродить с 50% HP.
    *   **Reroll (Рерол):** Не нравятся апгрейды? Посмотри рекламу — обнови список.
    *   **Double Gold:** В конце забега удвоить полученную валюту (для мета-прогрессии).

#### Интеграция Yandex SDK в Bevy (WASM):

Bevy компилируется в WebAssembly (`.wasm`). Чтобы общаться с Yandex SDK (который на JS), нужно использовать `wasm-bindgen`.

**Алгоритм:**
1.  В `index.html` подключаешь Yandex SDK.
2.  В Rust создаешь биндинги:

```rust
use wasm_bindgen::prelude::*;

#[wasm_bindgen]
extern "C" {
    // Функция из твоего JS, которая зовет Yandex SDK
    fn show_yandex_rewarded_ad(); 
}

// Система Bevy, которая триггерит рекламу
fn trigger_ad_system(mut events: EventReader<AdRequestEvent>) {
    for _ in events.read() {
        #[cfg(target_arch = "wasm32")]
        unsafe { show_yandex_rewarded_ad(); }
    }
}
```
3.  В JS (`index.html`) пишешь колбэк, который ставит игру на паузу (Bevy app pause) и показывает рекламу, а после успеха шлет обратно в WASM сигнал "начислить награду".

### 5. Потенциальные "Ловушки"

1.  **Производительность в вебе:**
    Bevy в WASM работает хорошо, но WebGL2 может капризничать на старых телефонах.
    *   *Совет:* Ограничь количество врагов (макс 200-300 активных). Если врагов больше, объединяй их в "Элитных" или просто не спавни новых. Не используй тяжелую физику.
2.  **Управление на мобилках:**
    Абстрактные игры часто страдают от того, что палец закрывает обзор.

3.  **Размер билда:**
    Яндекс любит быструю загрузку.
    *   *Совет:* Используй `wee_alloc`, включи LTO в `Cargo.toml`, прогони васм через `wasm-opt -Oz`.

Это отличная мысль. Чтобы игра не выглядела как очередной клон (слоп), но при этом укладывалась в MVP за неделю, нам нужно заменить скучные «+10% к урону» на **комбинаторику поведения**.

### 1. Концепция: Программирование снарядов (Logic Stack)

Вместо «оружия» у игрока есть **Потоки (Threads)**. Изначально доступен 1 поток (Main Thread).
Каждый поток выполняет цикл: `Spawn() -> Move() -> Event()`.

Прокачка — это добавление **Инструкций (OpCodes)** в этот цикл.

#### Как это выглядит в UI (Абстракция):
Игрок не выбирает "Топор", он выбирает функцию. Карточка апгрейда выглядит как кусок кода:
*   `> .fork(2)`
*   `> .reflect()`
*   `> .while(hp > 0)`

---

### 2. Техническая реализация в Bevy (ECS)

Чтобы не писать интерпретатор языка за неделю, мы используем **Component Composition**.

Снаряд (Projectile) — это сущность, у которой есть компонент `Program`, содержащий список перечислений (Enums).

```rust
// Логика поведения
#[derive(Component, Clone)]
struct Program {
    ops: Vec<OpCode>, // Список инструкций
    generation: u32,  // Для ограничения рекурсии (чтобы не крашнуть браузер)
}

#[derive(Clone, Copy)]
enum OpCode {
    MoveLinear(f32),      // Лети прямо
    MoveSine(f32, f32),   // Лети волной (амплитуда, частота)
    Orbit(f32),           // Крутись вокруг игрока
    
    // Триггеры (Hooks)
    OnHitSpawn(Box<OpCode>), // При попадании выполнить другое действие
    OnExpireExplode,         // При исчезновении взорваться
    
    // Модификаторы данных
    Fork(u8),             // Разделиться на N частей
    Pierce(u8),           // Пронзить N врагов
    Homing(f32),          // Наводиться (Pointer)
}
```

**Система (Bevy System)** просто бежит по снарядам и матчит (match) их `OpCode`. Это очень быстро и дешево.

### 3. Три архетипа билдов ("Киллер-фичи")

Вот готовые идеи билдов, которые звучат "по-хакерски" и создают визуальный хаос (идеально для Vampire Survivors жанра).

#### Билд №1: "Stack Overflow" (Рекурсия и Фракталы)
*Суть:* Снаряд при смерти порождает свои уменьшенные копии.
*   **База:** `Projectile` (обычная пуля).
*   **Апгрейд 1: `.fork(2)`** — При попадании пуля делится на 2 (угол +30/-30 градусов).
*   **Апгрейд 2: `.recursion(depth=3)`** — Дочерние пули наследуют способность делиться.
*   **Апгрейд 3: `Math.random()`** — Добавляет джиттер (дрожание) траектории, создавая "облако" шума.
*   **Визуал:** Вы стреляете одним квадратом, он попадает во врага, рассыпается на 2, те еще на 2. Экран заполняется лавиной геометрического шума.
*   **В Bevy:** Просто спавним новые энтити с `generation - 1`.

#### Билд №2: "Memory Leak" (Статичные зоны и DoT)
*Суть:* Снаряды не исчезают, а "зависают" в памяти, нанося урон.
*   **База:** `Beam` (Луч) или частица.
*   **Апгрейд 1: `GC.disable()` (Garbage Collection Off)** — Снаряды перестают исчезать за краем экрана или по таймеру. Вместо этого они застывают на месте.
*   **Апгрейд 2: `.persist(duration)`** — Застывшие снаряды живут 10 секунд, создавая "минное поле".
*   **Апгрейд 3: `.overflow()`** — Застывшие снаряды начинают расти в размерах, пока не лопнут.
*   **Визуал:** Игрок бегает и оставляет за собой шлейф "битых пикселей" (статичных квадратов), враги натыкаются на них и умирают.
*   **В Bevy:** Компонент `Velocity` удаляется, добавляется компонент `HazardZone` с коллайдером.

#### Билд №3: "Null Pointer" (Самонаведение и Глитчи)
*Суть:* Снаряды телепортируются или мгновенно находят цель.
*   **База:** Быстрые мелкие частицы.
*   **Апгрейд 1: `ptr.addressOf(enemy)`** — Самонаведение (Homing).
*   **Апгрейд 2: `transport.tcp()`** — Снаряд летит не плавно, а телепортируется рывками (блинк) к врагу. Игнорирует стены.
*   **Апгрейд 3: `try/catch`** — Если снаряд промахнулся, он возвращается к игроку и летит снова (бумеранг).
*   **Визуал:** Резкие дерганые линии, лазерные лучи, которые мгновенно соединяют игрока и врагов.

### 4. Реализация "Кода" в UI (MVP)

Не нужно делать сложный редактор. Сделайте **Слотовый инвентарь**, как в RPG, но назовите слоты терминами:

1.  **Init()** (Генератор): Сюда ставим "Тип снаряда" (Пуля, Луч, Аура).
2.  **Update()** (Поведение): Сюда ставим "Движение" (Синусоида, Рикошет, Наводка).
3.  **OnDestroy()** (Событие): Сюда ставим "Эффект смерти" (Взрыв, Спавн копии, Лужа).

**Пример геймплея:**
Игрок находит модуль `Recursion`.
*   Если он вставит его в слот `Update()` -> Снаряд будет двоиться во время полета.
*   Если он вставит его в слот `OnDestroy()` -> Снаряд разделится при попадании.

Это даст ту самую глубину **Theorycrafting**, но кодить вам придется всего одну систему, которая просто проверяет: `if slot_update == recursion { spawn_clone() }`.

### 5. Баланс и Монетизация этой системы

*   **Реклама (Rewarded):** "Overclocking".
    *   Предложите игроку "Разогнать CPU". Это ускоряет скорость выполнения всех программ (скорость полета и атаки) на 50% на 3 минуты.
    *   Или "Hotfix" — восстановить HP.
*   **Прогрессия:**
    *   Сначала у игрока 1 поток (оружие). За золото (мета-валюта) он покупает `Multi-threading` (второй слот под оружие).

### Почему это сработает как MVP?

1.  **Минимум ассетов:** Вам нужны только примитивы (Circle, Rect) и цвета. Вся красота в движении.
2.  **Простой код:** Логика `Enum` в Rust/Bevy пишется за 1-2 дня.
3.  **Маркетинг:** Скриншоты с кучей частиц и подписями в стиле кода (`.exe has stopped working`, `segfault attack`) выглядят стильно и привлекают технарей/геймеров.

Это концептуально сильнее, чем просто "+1 damage", и идеально ложится на архитектуру Bevy.

Для MVP на Bevy без подключения тяжелых физических движков (Avian/Rapier) лучше всего написать простую систему на базе **AABB** (если квадраты) или **Circle-Circle** (если круги/точки). Так как у тебя абстракция и "particles", круги подходят идеально — проверка дистанции самая дешевая операция.

Вот готовая, оптимизированная архитектура для Bevy 0.15.

### 1. Компоненты

Нам нужно пометить, кто есть кто, и задать размеры.

```rust
use bevy::prelude::*;

// Слой коллизии, чтобы не проверять всё со всем
#[derive(Component, Debug, Clone, Copy, PartialEq, Eq)]
pub enum CollisionLayer {
    Player,
    Enemy,
    Projectile,
    XpOrb,
}

// Радиус коллизии
#[derive(Component)]
pub struct Collider {
    pub radius: f32,
    pub layer: CollisionLayer,
}

// Событие столкновения
#[derive(Event)]
pub struct CollisionEvent {
    pub source: Entity, // Кто влетел (например, Пуля)
    pub target: Entity, // В кого влетели (например, Враг)
    pub layer_source: CollisionLayer,
}
```

### 2. Система обнаружения (Detection System)

Чтобы это работало быстро в браузере (WASM), мы не будем делать полный перебор всех со всеми ($N^2$). Мы разобьем проверки на конкретные пары.

Здесь используем `distance_squared`, чтобы избежать дорогой операции извлечения корня (`sqrt`).

```rust
pub fn collision_detection_system(
    mut events: EventWriter<CollisionEvent>,
    // Запрос 1: Снаряды
    projectiles: Query<(Entity, &GlobalTransform, &Collider), With<Projectile>>, 
    // Запрос 2: Враги
    enemies: Query<(Entity, &GlobalTransform, &Collider), With<Enemy>>,
    // Запрос 3: Игрок
    player: Query<(Entity, &GlobalTransform, &Collider), With<Player>>,
) {
    // 1. Проверка: Снаряды vs Враги
    // Итерируемся по снарядам, так как их обычно меньше или они чаще исчезают
    for (proj_entity, proj_tf, proj_col) in projectiles.iter() {
        let proj_pos = proj_tf.translation().truncate(); // Берем XY

        for (enemy_entity, enemy_tf, enemy_col) in enemies.iter() {
            let enemy_pos = enemy_tf.translation().truncate();
            
            // Сумма радиусов
            let collision_dist = proj_col.radius + enemy_col.radius;
            
            // Оптимизация: distance_squared
            if proj_pos.distance_squared(enemy_pos) < collision_dist * collision_dist {
                events.send(CollisionEvent {
                    source: proj_entity,
                    target: enemy_entity,
                    layer_source: CollisionLayer::Projectile,
                });
            }
        }
    }

    // 2. Проверка: Враги vs Игрок
    if let Ok((player_entity, player_tf, player_col)) = player.get_single() {
        let player_pos = player_tf.translation().truncate();
        
        for (enemy_entity, enemy_tf, enemy_col) in enemies.iter() {
             let enemy_pos = enemy_tf.translation().truncate();
             let collision_dist = player_col.radius + enemy_col.radius;

             if player_pos.distance_squared(enemy_pos) < collision_dist * collision_dist {
                 events.send(CollisionEvent {
                     source: enemy_entity,
                     target: player_entity,
                     layer_source: CollisionLayer::Enemy,
                 });
             }
        }
    }
}
```

### 3. Система "Анти-Стак" (Soft Collision)

Чтобы враги не слипались в одну точку (это выглядит дешево), нам нужна **Separation Logic**.
Мы не используем жесткую физику (отскок), мы просто немного расталкиваем их каждый кадр. Это создает эффект "толпы".

*Примечание: Полный перебор врагов друг с другом — тяжелая операция $O(N^2)$. Для MVP (до 300-500 врагов) Bevy это "съест" нормально. Если будет лагать — нужно внедрять Spatial Grid, но пока сделаем просто.*

```rust
pub fn enemy_separation_system(
    mut enemies: Query<(Entity, &mut Transform, &Collider), With<Enemy>>,
    time: Res<Time>,
) {
    let mut combinations = enemies.iter_combinations_mut();
    
    // Сила расталкивания
    let push_speed = 50.0; 

    while let Some([(e1_entity, mut e1_tf, e1_col), (e2_entity, mut e2_tf, e2_col)]) = combinations.fetch_next() {
        let p1 = e1_tf.translation.truncate();
        let p2 = e2_tf.translation.truncate();
        
        let dist_sq = p1.distance_squared(p2);
        let combined_radius = e1_col.radius + e2_col.radius;
        
        // Если они пересекаются (но дистанция не 0, чтобы не делить на ноль)
        if dist_sq < combined_radius * combined_radius && dist_sq > 0.001 {
            let dist = dist_sq.sqrt();
            let overlap = combined_radius - dist;
            
            // Вектор от 2 к 1
            let dir = (p1 - p2) / dist;
            
            // Расталкиваем обоих в разные стороны
            // Soft push: двигаем не мгновенно, а плавно через delta_seconds
            let move_vec = dir * overlap * push_speed * time.delta_secs();
            
            e1_tf.translation.x += move_vec.x;
            e1_tf.translation.y += move_vec.y;
            
            e2_tf.translation.x -= move_vec.x;
            e2_tf.translation.y -= move_vec.y;
        }
    }
}
```

### 4. Обработка событий (Геймплей)

Теперь ты просто подписываешься на `CollisionEvent` и пишешь логику.

```rust
pub fn handle_collisions(
    mut events: EventReader<CollisionEvent>,
    mut commands: Commands,
    mut enemy_query: Query<&mut Health, With<Enemy>>,
    // ... другие квери
) {
    for event in events.read() {
        match event.layer_source {
            CollisionLayer::Projectile => {
                // event.source -> Снаряд
                // event.target -> Враг
                
                // 1. Уничтожаем снаряд (если он не пробивной)
                commands.entity(event.source).despawn();

                // 2. Наносим урон врагу
                if let Ok(mut health) = enemy_query.get_mut(event.target) {
                    health.current -= 10.0;
                    if health.current <= 0.0 {
                        // Спавним партиклы смерти тут
                        commands.entity(event.target).despawn();
                    }
                }
            },
            CollisionLayer::Enemy => {
                 // Враг коснулся игрока -> Дамажим игрока
                 println!("Game Over logic check");
            },
            _ => {}
        }
    }
}
```

### Подробный разбор почему

#### Вариант 1: `Update` (Рекомендую)
Система запускается каждый кадр рендера (например, 60 или 144 раза в секунду).

```rust
fn movement_system(time: Res<Time>, mut query: Query<(&mut Transform, &Velocity)>) {
    // Важно: delta_seconds() может плавать, поэтому умножаем на скорость
    let dt = time.delta_seconds(); 
    for (mut transform, velocity) in query.iter_mut() {
        transform.translation += velocity.vec * dt;
    }
}
```
*   **Плюсы:** Идеальная плавность. Враг смещается ровно настолько, насколько прошло времени между отрисовками.
*   **Минусы:** Поведение немного зависит от FPS. Если FPS упадет до 5, `dt` станет огромным, и враг может пролететь сквозь стену (тут поможет Raycast или проверка шагов, но для Vampire Survivors это не критично).

#### Вариант 2: `FixedUpdate` (Для физики типа Rapier/Avian)
Система запускается фиксированное количество раз (по умолчанию в Bevy — 64 Гц).

```rust

fn fixed_movement_system(time: Res<Time>, mut query: Query<...>) {
    // time.delta_seconds() здесь ВСЕГДА константа (например, 0.0166)
    let dt = time.delta_seconds(); 
    // ...
}
```
*   **Плюсы:** Детерминизм. Поведение игры одинаковое на мощном ПК и слабом телефоне.
*   **Минусы (Критические для MVP):** **Джиттер (Тряска).**
    *   Твой монитор обновляется 144 Гц, а физика считается 60 Гц.
    *   Без сложной системы **интерполяции** (которую надо писать руками или тянуть плагины), спрайты будут двигаться рывками. Глаз это очень хорошо видит.

---

### Когда всё-таки нужен FixedUpdate?
Только для **логики геймплея**, не связанной напрямую с плавным перемещением спрайта:
1.  **Таймеры спавна волн:** Чтобы на лагающем телефоне волны не выходили медленнее.
2.  **Тики урона (DoT):** Чтобы яд наносил урон ровно раз в секунду.
3.  **Регенерация маны/HP.**

### Совет для Vampire Survivors (Пауза и Time Scale)
В Bevy есть `Time<Virtual>` и `Time<Real>`.
Для движения используй `Res<Time>` (это алиас для `Time<Virtual>`).

Почему это важно:
Когда игрок берет левел-ап или открывает меню, ты захочешь поставить игру на паузу.
Если ты используешь `Time<Virtual>`, тебе достаточно сделать:
```rust
mut time: ResMut<Time<Virtual>>
time.pause(); // Или time.set_relative_speed(0.0);
```
И все твои системы движения в `Update` застынут сами собой (так как `dt` станет 0). А UI и анимации меню (которые работают от `Time<Real>`) продолжат работать.

**Итог:** Двигай и проверяй коллизии в `Update`. Это стандарт для 2D аркад без тяжелой физики.
